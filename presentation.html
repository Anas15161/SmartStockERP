Pattern MVC 2 & Pilotage de Navigation

Jakarta Servlet 6.0 Jakarta Pages 3.1 CDI 4.0

Le pilotage est assuré par des servlets "Front-Controller" qui orchestrent les échanges entre la vue et le modèle.

Détail technique : L'utilisation de @Inject permet de déléguer l'instanciation des Repositories au conteneur Weld, assurant un couplage faible.
com.smartstock.erp.servlet.ProductServlet.java
@WebServlet(name = "ProductServlet", urlPatterns = {"/products"})
public class ProductServlet extends HttpServlet {

    @Inject
    private ProductRepository productRepository; // Injection CDI

    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        
        String action = request.getParameter("action");
        if ("lowstock".equals(action)) {
            request.setAttribute("items", productRepository.findLowStockProducts());
        } else {
            request.setAttribute("items", productRepository.findAll());
        }
        
        // Délégation à la vue JSP
        request.getRequestDispatcher("/products/list.jsp").forward(request, response);
    }
}

Couche Persistance & ORM Avancé

JPA 3.1 Hibernate 6.4 Criteria API

Gestion transparente du schéma Oracle XE via une cartographie objet-relationnelle rigoureuse.

Zéro JDBC : Toutes les requêtes sont générées par Hibernate. La relation @ManyToOne entre Produit et Fournisseur est gérée avec un FetchType.EAGER pour optimiser les performances lors de l'affichage des listes.
com.smartstock.erp.model.Product.java
@Entity
@Table(name = "PRODUCTS")
public class Product implements Serializable {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @NotNull @Size(min = 3)
    @Column(nullable = false, length = 100)
    private String label;

    @ManyToOne(fetch = FetchType.EAGER)
    @JoinColumn(name = "supplier_id")
    private Supplier supplier; // Relation bidirectionnelle

    @PrePersist @PreUpdate
    protected void onUpdate() {
        this.lastUpdated = LocalDateTime.now(); // Hook de cycle de vie
    }
}

Sécurité : Middleware & Filtrage HTTP

Jakarta Filter Session Tracking

Protection granulaire des ressources sensibles via un filtre d'authentification centralisé.

com.smartstock.erp.servlet.AuthenticationFilter.java
@WebFilter(urlPatterns = {"/products/*", "/suppliers/*", "/dashboard"})
public class AuthenticationFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) 
            throws IOException, ServletException {
        
        HttpServletRequest req = (HttpServletRequest) request;
        HttpSession session = req.getSession(false);

        // Bypass pour les ressources publiques et statiques
        if (session != null && session.getAttribute("user") != null) {
            chain.doFilter(request, response);
        } else {
            ((HttpServletResponse) response).sendRedirect(req.getContextPath() + "/login");
        }
    }
}

Interopérabilité : JAX-RS (REST API)

Jakarta REST 3.1 JSON-B 3.0 Jersey

Exposition de services Web pour l'intégration mobile et front-end découplé.

com.smartstock.erp.resource.ProductResource.java
@Path("/products")
@Produces(MediaType.APPLICATION_JSON)
public class ProductResource {

    @Inject
    private ProductRepository repository;

    @GET
    public List<Product> getJsonList() {
        return repository.findAll(); // Sérialisation automatique vers JSON
    }
}
